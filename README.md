# DOLE IDS Web

Laravel 13 + Inertia v3 (React 19) + Tailwind v4 application for the Department
of Labor and Employment supply/inventory system.

There are two ways to run it: **Docker** (recommended) or a **local** install.

---

## Prerequisites

- **Docker** + Docker Compose (for the Docker path), **or**
- **PHP 8.3+**, **Composer**, and **Node 22+** (for the local path — `setup.sh`
  can download a portable Node/Composer for you if missing).

---

## Run with Docker (recommended)

From the project root:

```bash
# 1. Create the env file (first time only)
cp src/.env.example src/.env

# 2. Build and start the stack
docker compose up -d --build
```

First boot takes ~1–2 min while dependencies install and Vite starts inside the
container. Then open:

> **http://localhost:8002**

`./setup.sh --docker` does the same build + up in one command.

### What's running

| Container | Purpose | Host port → container |
|---|---|---|
| `DoleIdsWeb.App` | PHP-FPM, Vite dev server, queue worker (via supervisor) | `5174 → 5174` (Vite) |
| `DoleIdsWeb.WebServer` | nginx (serves the app) | `8002 → 8004` |
| `DoleIdsWeb.Database` | PostgreSQL + PostGIS | `8833 → 5432` |

The app uses **SQLite** by default (`src/database/database.sqlite`). A
Postgres/PostGIS service is included — to use it, set these in `src/.env` and
restart the app container:

```env
DB_CONNECTION=pgsql
DB_HOST=dole-ids-web.database
DB_PORT=5432
DB_DATABASE=dole_ids_web
DB_USERNAME=dole
DB_PASSWORD=dole_pass
```

### First-time app setup (inside the container)

```bash
docker exec DoleIdsWeb.App php artisan key:generate
docker exec DoleIdsWeb.App php artisan migrate --seed
```

### Common Docker commands

```bash
docker compose up -d              # start
docker compose down              # stop
docker compose logs -f dole-ids-web.app   # tail app logs
docker restart DoleIdsWeb.App    # restart app (needed after PHP changes — see Notes)
docker exec -it DoleIdsWeb.App bash       # shell into the app
```

## Run locally (without Docker)

```bash
./setup.sh     # one-time: installs deps, builds frontend, creates SQLite DB + seeds
./serve.sh     # start Laravel on http://localhost:8000
./serve.sh --vite   # also start Vite (HMR) for frontend development
```

Open **http://localhost:8000**.

## Development

The frontend lives in `src/resources/js` (Inertia React pages under
`pages/<role>/`). In Docker, Vite runs automatically with hot-reload, so `.tsx`
changes appear on save (hard-refresh if needed).

Useful commands (run inside `src/`, or via `docker exec DoleIdsWeb.App ...`):

```bash
npm run dev          # Vite dev server (HMR)
npm run build        # production asset build
npm run types        # TypeScript check (tsc --noEmit)
vendor/bin/pint      # format PHP
php artisan test     # run the Pest test suite
```

## Notes / Troubleshooting

- **PHP changes don't take effect** → the container runs OPcache; restart it:
  `docker restart DoleIdsWeb.App`. (Route, controller, and other `.php` edits
  need this. Frontend `.tsx` edits do **not**.)
- **Page loads unstyled / blank** → the Vite dev server or its `public/hot` file
  is stale. `docker restart DoleIdsWeb.App`, then hard-refresh (Ctrl+Shift+R).
- **"Unable to locate file in Vite manifest"** → run `npm run build` (or make
  sure the Vite dev server is running).
- **Wayfinder route helpers** (`@/routes`, `@/actions`) are auto-generated by the
  Vite plugin. Don't run `php artisan wayfinder:generate` by hand — it drops the
  form variants this project uses. Let Vite regenerate them.
